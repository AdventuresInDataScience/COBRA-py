**Findings**
1. Critical: Exit rule combination is implemented as AND, but design requires ANY exit rule to trigger exit (OR).
- Spec says entry uses conjunction, while exits fire when any exit rule is true in COBRA-py_Summary_Design.md.
- Current implementation applies AND reduction for all rule groups in rules.py, and that is used for exits in engine.py.
- Impact: exits can be suppressed, changing trade duration, drawdown, and return metrics materially.

2. Critical: Optimizers are not optimizing policy parameters directly; they optimize a sampled integer seed.
- Nevergrad optimizes one scalar integer in nevergrad_runner.py, then maps that to a sampled config in nevergrad_runner.py.
- TPE does the same via sample_seed in tpe_runner.py.
- DEHB native is forced to 1 dimension in dehb_runner.py and then converted to a sample seed in dehb_runner.py.
- This conflicts with the documented conditional ConfigSpace-first design in COBRA-py_Implementation_Plan.md, and means logic options are only indirectly hit by random sampling rather than explicitly searched.

3. High: Ulcer objective can reward invalid/sentinel metrics.
- Sentinel bad metric is -999.0 in metrics.py.
- Objective for ulcer directly minimizes ulcer_index in function.py.
- If ulcer_index is sentinel -999, minimization treats it as excellent (very low risk), which is the opposite of intended invalid-strategy penalization.

4. High: Search space exposes fewer logic options than documented.
- Documented archetypes include comparison, crossover, band_test, pattern, stat_test, derivative in COBRA-py_Implementation_Plan.md.
- Actual sampler only chooses comparison/crossover, plus band_test for bb in space.py.
- Documented indicator universe includes keltner/donchian/adx/stoch/psar/cci/roc in COBRA-py_Summary_Design.md, but selectable rule indicators are limited in space.py.

5. Medium: Nevergrad reproducibility seed is not applied to optimizer RNG.
- Design notes require explicit random_state seeding for Nevergrad in COBRA-py_Summary_Design.md.
- run_nevergrad accepts seed in nevergrad_runner.py but does not set optimizer/parametrization random state.
- Impact: same run config may not be reproducible.

6. Medium: CLI walk-forward ignores selected optimizer and always uses DEHB.
- Walk-forward branch starts in cli.py.
- Fold optimizer is hardcoded to run_dehb in cli.py.
- Impact: if user selected nevergrad or tpe, fold behavior is inconsistent with top-level optimization.
